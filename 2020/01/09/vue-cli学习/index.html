vue-cli 学习

1.vue-cli 安装

  • 先确保电脑已经配置 node 环境
  • 之后通过命令 npm install -g @vue/cli 来安装 vue-cli 到全局配置
  • 安装之后，可以通过命令 vue --version 来确定是否成功安装了 vue-cli 并确定版本号

创建一个项目

  • cmd 窗口下，通过 cd 命令，找到想要创建项目的文件夹位置 ( 或到文件夹目录下通过 shift + 右键 选择打开 powershell 窗口 )

  • 输入命令 vue create <filename> 来生成一个新项目， <filename>为项目名字

  • 之后会出现两个选项，default (babel, eslint) 和 Manually select features 其中 default 为默认选中 babel 和 eslint 两个配置；另一个则为自己选择配置

  • 如想自己配置，则选中 Manually select feature 按回车，选择想要的配置，按空格选中或取消选中，之后，按回车进行下一步操作

  • 之后，会弹出选项询问把刚刚选中的配置文件放在哪里

    ​ In dedicated config files 独立文件放置

    `In package.json` 放入 package.json 文件中

    这里推荐放入 package.json 文件中

  • 最后选择是否作为预设，我选择 N，在经过一段时间后，项目创建完成

项目运行

  • 在创建完项目后根据提示 cd 到项目目录下
  • 执行命令 npm run serve 来运行项目到端口上，默认为 8080，如果被占用，则会变为 8081，依次类推
  • 当开始运行后，通过在端口输入 ip 地址来进行切换到选择的界面 http://localhost:8080/
  • 此时，可以看到 vue-cli 的主页

项目目录分析

  • 在自动生成项目后，你会发现项目中有很多个文件
  • 其中 README.md 自然就是对 vue-cli 的一些基本介绍
  • package.json 中则存放了本项目的一些依赖和刚刚选中的基本配置
  • base.conf.js用于对 ES6 等语法进行处理
  • gitignore 在上传时告诉 git 哪些是不用上传的
  • public 中存放的则是公司图标，以及主页
  • node_modules 中存放的是项目的依赖包
  • src 是我们开发过程中需要用到的一个文件夹,里面又包含了几个文件及文件夹
    • assets 用于存放一些静态的资源，如 js、css、以及图片，都存放在里面
    • component 中存放我们之后要用的一些组件，其命名方式一般为驼峰命名法，且第一个单词的第一个字母也大写，组件由三部分组成
      • template：类似于 html 但只能有一个主节点，如出现两个或以上，则会报错
      • script：用于存放 js 代码
      • style：用于存放 css 代码
    • app.vue 为根组件，后续会讲到
    • main,js 为逻辑入口文件,其中的 #app 对应着 index 的app
